# 워터 팩토리 (물의 상태변화 공장) — 설계 문서

- **작성일**: 2026-04-17
- **대상**: 초등학교 4학년 1학기 과학 2단원 "물의 상태 변화"
- **플레이 환경**: 삼성 갤럭시탭 Chrome (가로모드), 개인 플레이
- **장르**: 2D 레트로 픽셀아트 타이쿤 아케이드 (Phaser.js)

---

## 1. 목적과 범위

### 1.1 학습 목적
학생이 반복 플레이를 통해 **물의 세 가지 상태(고체·액체·기체)** 와 **4가지 상태 변화**를 교과서 용어로 자연스럽게 익힌다.

교과서 용어 (4-1-2 핵심개념 JSON 기준):
- 고체(얼음) / 액체(물) / 기체(수증기)
- **녹음** (고체→액체), **얼음/얼다** (액체→고체)
- **끓음** (액체→기체, 가열), **응결** (기체→액체)
- "응고/융해/기화/액화"는 교과서에 **없음** — 중학교 용어라 배제.

### 1.2 게임 목적
재미 최우선. 학생이 쉬는 시간·자율 학습·가정 학습 등 자율적으로 반복 플레이할 만큼 몰입도 있게 설계.

### 1.3 범위 (MVP)
- 싱글 플레이 웹게임, 서버 없음
- 단일 공장 화면, 주문 6종, 목숨 5개
- 정적 호스팅 (OCI + nginx 서브경로)
- 자동 테스트는 GameState 순수 로직에 한정, UI는 수동 체크리스트

**범위 외** (향후 V2):
- 멀티플레이, 업그레이드 시스템, 보스
- 상태변화별 상세 리포트, 플레이 히스토리
- 배경음악, PWA 오프라인, 학급 리더보드

## 2. 게임 루프

### 2.1 핵심 루프
```
[주문 받음] → [현재 상태 확인] → [🔥/❄️ 변환] → [📦 출하]
           → [정답/오답 판정] → [점수 반영] → [다음 주문]
```

### 2.2 파라미터
| 항목 | 값 |
|---|---|
| 동시 주문 | 항상 1건 |
| 초기 제한시간 | 10초 |
| 제한시간 감소 | 점수 200점당 0.5초 ↓ |
| 최소 제한시간 | 4초 |
| 정답 점수 | 100 + (남은 시간 × 10) |
| 오답/시간초과 | 목숨 -1 |
| 초기 목숨 | 5 |
| 게임 종료 | 목숨 0 |

### 2.3 정답/오답 판정
- **정답**: 주문의 목표 상태와 출하 시 가마솥 상태 일치
- **오답**: 상태 불일치 상태로 📦 출하
- **시간초과**: 10초(점진 감소) 내 📦 미출하

## 3. 상태, 전이, 주문

### 3.1 상태 3종
- 🧊 **고체** (얼음)
- 💧 **액체** (물)
- 💨 **기체** (수증기)

### 3.2 상태 전이
- 🔥 버튼 = 한 단계 상승 (고체→액체→기체)
- ❄️ 버튼 = 한 단계 하강 (기체→액체→고체)
- 경계 상태(고체에서 ❄️, 기체에서 🔥)는 버튼 회색 비활성 + 탭 시 짧은 흔들림 피드백
- 변환 애니메이션 **0.5초** — 중엔 모든 버튼 비활성 (연타 방지)

### 3.3 주문 6종 (MVP)
| 상태 | 주문 |
|---|---|
| 🧊 고체 | 아이스크림, 얼음 큐브 |
| 💧 액체 | 수족관 물, 시원한 음료수 |
| 💨 기체 | 가습기 수증기, 찜질방 스팀 |

### 3.4 시작 상태 랜덤화
- 매 주문마다 가마솥 초기 상태를 고체/액체/기체 중 랜덤 선택
- 결과: 바로 출하 / 1단계 변환 / 2단계 변환이 자연스럽게 섞여 4가지 상태 변화 모두 노출

### 3.5 상태 변화 완료 팝업 (교과서 용어)
변환 성공 순간 가마솥 위에 0.8초 노출 후 페이드아웃.

| 방향 | 팝업 문구 |
|---|---|
| 고체 → 액체 | `"✨ 녹음 성공! 얼음 → 물"` |
| 액체 → 기체 | `"✨ 끓음 성공! 물 → 수증기"` |
| 기체 → 액체 | `"✨ 응결 성공! 수증기 → 물"` |
| 액체 → 고체 | `"✨ 얼음 완성! 물이 얼었어요"` |

> "증발"은 게임 메커닉상(🔥 가열) 등장하지 않음. 교실 지도 시 선생님이 "증발도 있다"고 보완.

## 4. UI 레이아웃

### 4.1 전체 컨셉
**공장 뷰 통합형** — 화면 전체가 픽셀아트 공장 내부. UI 요소가 공장 세계에 녹아있음.

### 4.2 해상도
- 기본 **1280×720 (16:9)**
- Phaser `Scale.FIT` — letterbox로 화면 비율 보장
- `pixelArt: true`로 정수 배율 확대
- 세로모드 진입 시: "가로로 돌려주세요" 안내 화면

### 4.3 화면 구성
```
┌──────────────────────────────────────────┐
│ [점수 1,240]   [❤️❤️❤️❤️❤️]    [⏱ 7.3s]  │  ← 공장 상단 파이프/계기판에 HUD 녹여냄
│                                          │
│ ┌────┐                                   │
│ │주문│          ┌──────────┐             │
│ │🍦  │          │          │             │
│ │아이│          │  🧊/💧/💨 │             │  ← 중앙 가마솥 (상태별 스프라이트)
│ │스크│          │  (가마솥) │             │
│ │림  │          └────▲─────┘             │
│ └────┘           🔥 불꽃 (🔥 누를 때)     │
│  ↑ 좌측          ❄️ 서리 (❄️ 누를 때)     │
│  압정 보드                                │
│          ┌──────┐┌──────┐┌──────┐        │
│          │  🔥  ││  ❄️  ││ 📦   │        │  ← 하단 기계 조작 패널
│          └──────┘└──────┘└──────┘        │
│  [공장 바닥/파이프/환풍구 장식]          │
└──────────────────────────────────────────┘
```

### 4.4 배경 장식 (기본 수준, 스프라이트 약 20개)
- 벽, 바닥 타일
- 파이프 (상단), 계기판 (상단), 환풍구 (우측 상단)
- 중앙 가마솥 1대 (기계 형태)
- 장식은 **정적** (움직이는 연출은 V2)

### 4.5 가마솥 연출
- 금속 가마솥 스프라이트, 상단 개방
- 상태별 내용물: 얼음 블록 (고체) / 찰랑이는 물 (액체) / 빈 솥 + 상단 수증기 (기체)
- 🔥 누를 때: 솥 아래 불꽃 애니메이션, 얼음→물 녹는 프레임, 물→수증기 끓어오르는 프레임
- ❄️ 누를 때: 솥 주변 서리 이펙트, 수증기 뭉쳐 물방울 되는 프레임, 물 표면 얼어가는 프레임
- 상태 변화 성공 시 3.5의 팝업 표시

## 5. 타이틀 씬 (스토리 + 게임 방법)

### 5.1 스토리 컨셉
**신입 공장장** — "오늘부터 당신은 상태변화 공장의 신입 공장장! 주문 받아 물을 변환해 납품하자."

### 5.2 슬라이드 3장
사용자가 탭할 때마다 다음 슬라이드로. 각 슬라이드 우측 상단에 [건너뛰기] 버튼.

| # | 내용 | 예시 대사 |
|---|---|---|
| 1 | 스토리 소개 | "첫 출근 날! 손님들의 주문을 받아 물을 세 가지 상태로 변환해 납품하자." |
| 2 | 조작 방법 | "🔥 온도 올리기 · ❄️ 온도 내리기 · 📦 출하" (각 버튼 예시 이미지) |
| 3 | 목표 | "목숨 5개! 시간 내 올바른 상태로 납품하면 점수, 실수하면 목숨 -1." |

## 6. 게임 오버 & 리포트

### 6.1 트리거 연출
목숨이 0이 된 순간:
1. 공장 알람 🚨 효과음 (사운드 ON 시)
2. 화면 전체 붉은 오버레이 깜빡임 1초
3. 페이드아웃 → 게임 오버 오버레이

### 6.2 게임 오버 오버레이 (미니멀 리포트)
```
╔════════════════════════╗
║       GAME OVER        ║
║                        ║
║   최종 점수: 1,240     ║
║   완수 주문: 12건      ║
║                        ║
║   🏆 최고 점수: 1,450  ║
║                        ║
║      [🔁 다시하기]     ║
╚════════════════════════╝
```
- 이번 점수가 최고 점수를 넘었을 경우 `🎉 NEW RECORD!` 배지 표시 + localStorage 업데이트
- 버튼은 **[🔁 다시하기]** 1개만. "처음으로" 버튼 없음 (학생이 브라우저 닫으면 됨)
- 다시하기 = GameScene.restart()

## 7. 기술 구조

### 7.1 씬 구성
```
BootScene
  └─ 에셋 프리로드 (스프라이트, 효과음) + 로딩 바
  └─ 완료 시 → TitleScene

TitleScene
  └─ 3슬라이드 스토리/방법 + [▶ 시작] + 최고 점수 표시
  └─ [▶ 시작] → GameScene

GameScene
  └─ 플레이 루프 (주문 생성 → 입력 처리 → 출하 판정)
  └─ 목숨 0 도달 → 게임 오버 오버레이 (같은 씬 내 그룹 오브젝트)
  └─ [🔁 다시하기] → this.scene.restart()
```

### 7.2 상태 관리

**`src/state/GameState.ts`** (순수 로직 클래스)
- 필드: `score`, `lives`, `currentOrder`, `currentCauldronState`, `remainingTime`, `completedCount`
- 메서드:
  - `startGame()` — 상태 초기화
  - `generateNextOrder()` — 주문 6종 중 랜덤, 시작 상태 랜덤
  - `applyHeat()`, `applyCool()` — 상태 전이
  - `ship()` — 판정, 점수/목숨 반영, `{ correct: boolean, points: number, transition: string | null }` 반환
  - `tick(deltaMs)` — 시간 감소, 만료 시 목숨 차감
  - `isGameOver()` — `lives === 0`
- 순수 로직이라 **Phaser 없이 유닛 테스트 가능**

**Phaser Registry** (씬 간 공유)
- `highScore` — 타이틀·게임오버에서 읽고, 게임오버 시 localStorage 동기화
- `soundEnabled` — 🔊 토글 상태

**GameScene**
- `GameState` 인스턴스를 필드로 보유
- Phaser 이벤트(버튼 탭, 업데이트 틱) → GameState 메서드 호출 → 반환값 기반으로 렌더링 갱신
- 렌더링·애니메이션·입력만 담당

### 7.3 에셋 전략
- 소스: **Kenney.nl CC0 팩** (구현 단계에서 개발자가 선생님께 후보 링크 제시 → 승인 후 `curl`로 자동 다운로드)
- 후보 방향: 픽셀 공장/기계류, 픽셀 UI 버튼 팩, 픽셀 인터페이스 사운드 팩
- `public/assets/LICENSE.txt`에 라이선스 정보 명시
- 팩에 없는 오브젝트(예: "가마솥")는 비슷한 통/냄비 대체 스프라이트 사용, 필요 시 색 교체

### 7.4 localStorage
| 키 | 값 |
|---|---|
| `waterFactory.highScore` | number (최고 점수) |

초기 로드 시 존재하지 않으면 0으로 초기화.

### 7.5 사운드
- 효과음만, **기본 음소거**
- 화면 우측 상단에 🔊/🔇 토글
- 효과음 종류: 버튼 탭, 변환 성공, 출하 정답, 출하 오답/시간초과(알람), 게임오버

}
