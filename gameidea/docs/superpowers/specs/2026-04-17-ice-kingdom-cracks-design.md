## 교육 목표 요약

본 게임은 **초등학교 4학년 1학기 과학 "물의 상태 변화" 단원**의 두 핵심 개념을 내재화시키는 것을 목표로 한다.

1. **물이 얼면 부피가 늘어난다** (부피 팽창)
2. **상태가 변해도 무게는 변하지 않는다** (무게 보존)

특히 2번은 4학년 학생이 자주 드러내는 오개념("부피가 늘어났으니 무게도 늘었을 것이다")을 타격하는 데 집중한다. 저울 메커니즘이 게임 전체에 반복 등장하며 개념을 여러 상황에서 재확인하도록 설계한다.

## 1. 컨셉과 세계관

### 한 줄 정의

> 물과 얼음을 다뤄 균열 난 얼음왕국의 색을 되돌리는, 30~40 스테이지의 턴제 격자 퍼즐 게임.

### 서사 아웃라인

- **배경**: 오랜 겨울로 얼어붙은 얼음왕국. 어느 날 왕국 전체에 금이 가기 시작하며 왕국의 **색깔이 빠져나간다**.
- **주인공**: 여왕의 어린 정원사(아이 캐릭터). 얼음꽃 조각을 모아 왕국 각 지역의 색을 복원해야 한다.
- **메타 서사**: 플레이어가 챕터를 클리어할 때마다 세계 지도에 **파스텔 색이 차오르는 연출**이 나온다.
- **톤**: 멜랑콜리 + 따뜻함. 레퍼런스: *Gris* (감성) × *Baba Is You* (퍼즐 문법).

### 챕터 구조 (4챕터)

| Ch | 지역 | 학습 초점 | 새 메커니즘 |
|----|------|----------|-----------|
| 1 | 얼어붙은 정원 | 물↔얼음 부피 팽창 | 물샘, 얼리기, 얼음 밀어냄 |
| 2 | 저울의 탑 | 무게 불변 (오개념 1차) | 양팔저울 균형 맞추기 |
| 3 | 모닥불 광장 | 팽창 + 무게 복합 | 모닥불 자동 녹임, 게이트 저울 |
| 4 | 색이 돌아온 왕궁 | 종합 · 피날레 | 시소저울, 3요소 결합 퍼즐 |

### 챕터 엔딩 연출

- 짧은 일러스트 컷신 (8~15초)
- 해당 지역의 색이 회색에서 파스텔로 피어오르는 장면
- 여왕/정원사의 짧은 대사 1~2줄 — 해당 챕터 개념을 **암시적으로 복습**
  - Ch2 엔딩 대사 예: "얼음은 커졌어도, 무게는 여왕의 한숨만큼 가볍구나."

## 2. 핵심 메커닉 (턴 그래머)

### 기본 구조

- **탑다운 격자 맵** (스테이지마다 가변, 보통 7×7 ~ 12×12)
- **턴제**: 플레이어가 행동 1개 선택 → 세계가 1턴 진행 (모닥불 자동 녹임 등 환경 효과 포함)

### 플레이어 턴당 1개 행동 선택

| 행동 | 조작 | 턴 소비 | 효과 |
|------|------|:------:|------|
| **이동** | 인접 칸 탭 | O | 주인공 한 칸 이동. 얼음 위로 이동 시 **벽까지 미끄러짐** |
| **물 붓기** | 물샘 인접에서 빈 칸 탭 | O | 해당 빈 칸에 물 1칸 생성 (인접 범위 한정) |
| **얼리기** | 인접 물 칸 탭 + 방향 선택 | O | 물 1칸 → 얼음 2칸 (지정 방향으로 팽창) |
| **녹이기** | 인접 얼음 칸 탭 | O | 얼음 → 물 1칸 (팽창 자리는 빈 칸) |
| **Undo** | 버튼 | ✕ | 이전 상태 복원 (무제한) |
| **재시작** | 버튼 | ✕ | 스테이지 초기 상태로 |

> 턴을 소비하는 행동만 **모닥불 자동 녹임 등 환경 효과**를 1회 트리거한다. Undo/재시작은 환경 효과도 함께 이전 상태로 복원된다.

### 핵심 규칙

**1) 얼리기 팽창 규칙**
- 지정한 방향의 인접 칸이 **비어있어야** 얼릴 수 있다.
- 밀어낼 수 있는 물체(나무상자 등)가 있으면 한 칸 밀려난다.
- 팽창 방향에 **벽 또는 밀 수 없는 물체**가 있으면 얼리기는 **실패**(피드백: 아이콘 빨간 표시).
- 팽창 방향에 **물샘 · 모닥불 · 주인공**이 있으면 얼리기 **실패** (이들은 밀 수도, 덮을 수도 없다).
- 팽창 방향에 **또 다른 물 칸**이 있으면 얼리기 **실패** (물은 밀지 않는다 — 의도적 제약으로 퍼즐 단순화).

**2) 무게 불변의 법칙** (게임의 학습 심장부)
- 물 1칸 = 얼음 2칸 = **동일한 무게**.
- 양팔저울 위에 물 1칸 vs 건너편 얼음 2칸 올려도 **저울은 균형**.
- 게이트 저울: 얼려도 무게가 같아 더 무거워지지 않는다.
- 시소저울: 부피가 변해도 기울기는 변하지 않는다.

**3) 얼음 위 미끄러짐**
- 얼음 타일 위로 주인공이 "이동"으로 진입하면 **그 방향으로 계속 미끄러진다**.
- 벽·물체 또는 얼음이 끊긴 칸에 닿으면 정지.
- 미끄러지는 동안은 1턴으로 간주한다.

**4) 모닥불 자동 녹임**
- 모닥불 타일의 4방향 인접 얼음은 **매 턴 자동으로 녹아** 물이 된다.
- Ch3 이후의 시간 압박 연출에 사용 ("모닥불 근처 얼음 통로, 녹기 전에 건너가자").
- **예외**: 주인공이 현재 밟고 있는 얼음은 녹지 않는다 (플레이어 녹이기 액션도 주인공 자기 발밑 얼음은 선택 불가). 주인공이 이동해서 벗어난 뒤부터 녹기 시작한다.

### 스테이지 클리어 조건

- 맵에 흩어진 **얼음꽃**을 수집 (필수 1개 + 선택 1~2개)
- 주인공이 **출구 타일**에 도달
- 선택 꽃 포함 여부에 따라 **별 1/2/3** 평가 (회수 플레이 유도)

### 실패 조건

없음. 막히면 Undo 또는 재시작. 4학년 학습 심리 부담 제거.

## 3. 타일 · 오브젝트 · 레벨 데이터 포맷

### 지형 타일

| 기호 | 이름 | 설명 |
|------|------|------|
| `#` | 벽 | 통과 불가, 얼음 팽창이 밀어낼 수 없음 |
| `.` | 빈 바닥 | 이동 가능, 물/얼음 배치 가능 |
| `W` | 물 | 주인공 통과 불가. 얼리면 얼음이 됨 |
| `I` | 얼음 | 진입 시 미끄러짐. 녹이면 물이 됨 |
| `S` | 물샘 | 인접 빈 타일에 "물 붓기" 가능. 얼릴 수 없음 |
| `B` | 모닥불 | 4방향 인접 얼음을 매 턴 자동 녹임 |

### 오브젝트

| 기호 | 이름 | 설명 |
|------|------|------|
| `P` | 주인공 시작점 | 스테이지 시작 위치 |
| `F` | 필수 얼음꽃 | 수집해야 출구 열림 |
| `f` | 선택 얼음꽃 | 회수 실력 보상 (별점) |
| `E` | 출구 | 필수 꽃 수집 시 활성화 |
| `X` | 나무상자 | 얼음 팽창이 밀 수 있음 |
| `R` | 바위 | 밀 수 없음. 옆에서 얼리면 **갈라짐** (Ch1 보상 연출) |

### 저울 타일 (3형태)

| 기호 | 형태 | 등장 챕터 |
|------|------|----------|
| `L` `=` `R` | 양팔저울 (좌판 · 축 · 우판, 3칸 연속) | Ch2, Ch4 |
| `G` | 게이트 저울 (단판 + 연결된 문) | Ch3, Ch4 |
| `~` | 시소저울 (2칸짜리 발판) | Ch4 |

**저울 판 타일 규칙**
- `L`, `R`, `G`, `~` 타일은 **바닥 역할**을 한다 — 그 위에 물·얼음·상자·주인공이 올라갈 수 있다.
- 저울 무게 계산은 판 타일 **위에 올라간 오브젝트**의 합으로 결정된다.
- 물 1칸 = 얼음 2칸 = 상자 1칸 = 주인공 1칸 = **모두 무게 1 단위**로 간주 (4학년 인지 단순화).
- 얼음이 판 위에서 팽창하여 판 밖 칸까지 걸쳐도, 무게는 **판 위 영역에 걸친 얼음 칸 수**로만 집계. (즉 판 위에 1칸, 판 밖에 1칸 걸친 얼음은 판에 1 단위로만 계산.)
  - 단, **무게 불변의 학습 효과를 위해** 기본 교재 스테이지에서는 얼음 팽창이 판 안에서 완전히 일어나도록 디자인한다. 판 밖 걸침은 Ch4 응용 스테이지에서만 등장.

```yaml

name: "균형의 춤"

  #########
  #.......#
  #..P....#
  #....W..#
  #..SW...#
  #.L=R..F#
  #.......#
  #......E#
  #########
required_flowers: 1

    condition: "equal"

```

### 포맷 선택 근거

1. **ASCII 그리드**는 Claude가 직접 읽고 쓰기에 최적 — 초안 대량 생성 가능.
2. 스테이지당 **1~2 KB** → 버전 관리 · 공유 용이.
3. 별도 레벨 에디터 불필요 (개발 후반 여유되면 추가 가능).
4. **핫리로드**: 개발 중 파일 저장 즉시 반영.

## 4. 챕터 · 레벨 구성과 난이도 곡선

### 전체 스테이지 총괄 (목표 36 스테이지 + 비밀 스테이지 2~4)

| 챕터 | 제목 | 스테이지 수 | 새 메커니즘 | 핵심 학습 |
|------|------|-----------|------------|----------|
| 1 | 얼어붙은 정원 | 8 | 물샘, 얼리기, 팽창, 미끄러짐, 상자/바위 | 물↔얼음 상태 변화, 부피 팽창 |


### 챕터 4 — 색이 돌아온 왕궁 (피날레)

```
4-1  "시소의 방"          시소저울 튜토
4-2  "움직이는 발판"      시소 + 미끄러짐 결합
4-3  "여왕의 시험"        시소 + 저울 + 모닥불 전부 등장
4-4 ~ 4-8  종합 퍼즐 5개  메커니즘 자유 조합, 맵 크기 확대
                         (구체 설계는 Phase 3 중 Ch1~3 파일럿 피드백 반영하여 확정)
4-9  "왕궁의 심장"        최고 난도 (15~20턴)
4-10 "색이 피어난다"      엔딩 서사 중심 스테이지 → 최종 엔딩 컷신
```

### 난이도 곡선 4축

스테이지 난이도는 다음 4개 축으로 설계한다:

1. **맵 크기**: 5×5 → 12×12
2. **필수 메커니즘 개수**: 1개 → 5개 (얼리기 + 저울 + 모닥불 + 미끄럼 + 상자)
3. **제약 조건**: 없음 → 있음 (모닥불 타이밍, 좁은 공간)
4. **수학적 복잡도**: 무게 계산 1회 → 다중 저울 동시 만족

### 좌초 방지 UX

- 스테이지를 **3분 이상** 클리어하지 못하면 힌트 버튼이 활성화됨 (선택 클릭).
- 힌트는 "첫 수 방향" 한 줄만 제공 — 답 자체는 노출하지 않음.
- 3회 이상 클리어 실패 시 **챕터 건너뛰기 옵션**을 설정 메뉴에서 제공 (교사가 학생별로 허용/제한).

### 튜토리얼 UX

- 새 메커니즘이 등장하는 스테이지 시작 시 **주인공 머리 위 말풍선** 1~2줄.
- 예: 3-1 "어? 모닥불 옆 얼음이 혼자 녹고 있어!"
- 교과서적 설명은 배제, **관찰 유도**를 원칙으로.
- 한 번 본 말풍선은 기억하여 다시 표시하지 않음.

### 데이터 흐름

```
플레이어 탭
    ↓
InputHandler (StageScene)
    ↓
TurnEngine.executeAction(action)
    ├→ Grid 상태 변경 (순수 계산)
    ├→ UndoStack.push(prev state)
    ├→ 환경 업데이트 (모닥불 자동 녹임)
    └→ 승리 조건 체크
    ↓
entities/*Sprite 재렌더 + 애니메이션 tween
    ↓
UI 갱신 (HUD, 꽃 카운트)
```

### 핵심 설계 원칙

**1) 로직/렌더 분리**
- `core/` 모듈은 Phaser를 import하지 않는다.
- `Grid`, `TurnEngine`, `WeightRules`는 **순수 함수/클래스**로 구성 → 유닛 테스트 용이.
- 레벨 디자인 검증 스크립트(솔버)를 Node 환경에서 바로 실행 가능.

**2) 레벨 YAML = Single Source of Truth**
- 스테이지 추가/수정은 파일 1개 편집으로 완결.
- Claude에게 "Ch3 중급 스테이지 YAML 3개 생성" 식으로 초안 요청 가능.
- 핫리로드로 즉시 반영.

**3) Undo = 상태 스냅샷 방식**
- 매 턴 `Grid.clone()` → `UndoStack`에 푸시.
- 격자 크기가 작으므로 메모리 부담 없음 (~1 KB/턴).

**4) 저장 = localStorage 단순 JSON**
- 싱글플레이이므로 서버 없음.
- 스키마: `{ completedStages: string[], starsByStage: Record<string, 1|2|3> }`.

### 사운드 · 애니메이션

- **얼리기**: 결빙 particle + "찡" 짧은 효과음
- **녹이기**: 수증기 particle + "스르르" 효과음
- **미끄러짐**: 주인공이 관성 있게 미끄러지는 tween (~0.3초)
- **모닥불**: 일렁이는 idle 애니 + 인접 얼음에서 **하얀 증기** 피어오름
- **챕터 엔딩**: 해당 지역 타일이 회색 → 파스텔로 **color tween**, BGM에 피아노 한 줄

### 외부 의존성

- `phaser` (3.x)
- `js-yaml` (레벨 파서)
- `typescript` + `vite` (빌드)
- `vitest` (core 로직 유닛 테스트)
- 정적 배포 타겟: itch.io HTML5 + GitHub Pages

## 6. UI/UX

### 씬 플로우

```
[BootScene]  → 로딩바 (2~5초)
    ↓
[TitleScene] → "얼음왕국의 균열" 로고 + 시작/이어하기
    ↓
[ChapterSelectScene] ← 월드맵: 4개 지역 (복원 정도에 따라 색)
    ↓
[CutsceneScene] ← 챕터 시작 컷신 (짧은 일러스트 + 대사)
    ↓
[StageScene] ← 실제 퍼즐 플레이 (가장 오래 머무는 씬)
    ↓ (클리어)
[StageCompleteModal] ← 별점 + 다음 스테이지
    ↓ (챕터 끝)
[CutsceneScene] ← 챕터 엔딩 컷신 (색 복원 연출)
    ↓ (마지막 챕터)
[EndingScene] ← 최종 엔딩
```

### 인게임 HUD 레이아웃 (태블릿 가로모드)

```
┌────────────────────────────────────────────────┐
│  [←뒤로]  2-3 균형의 춤          [설정⚙]      │  ← 상단바
│                                                │
│                ┌──────────┐                    │
│                │          │                    │
│                │  격자 맵  │  ← 중앙 70%       │
│                │          │                    │
│                └──────────┘                    │
│                                                │
│  [🔄재시작]  [💡힌트]    꽃 🌸1/2    [↶Undo]  │  ← 하단 액션바
└────────────────────────────────────────────────┘
```

- 격자 맵이 화면의 **약 70%**를 차지 (가장 시선 집중).
- 상단·하단 바의 터치 타겟은 **최소 64 px** (4학년 손가락 여유).
- 행동 선택은 **모드리스**: 맵에 직접 탭해서 결정.

### 주요 인터랙션 상세

**1) 맵 탭으로 행동 자동 결정 (모드 없음)**
- 주인공 인접 빈 칸 탭 → 이동 (물샘 근처 빈 칸이면 "물 붓기/이동" 선택 팝업)
- 물 칸 탭 → 얼리기 방향 선택 팝업
- 얼음 칸 탭 → 녹이기 확인 (인접한 경우에만 활성화)

**2) 얼리기 방향 선택**
- 물 칸 탭 시 해당 칸 위에 4방향 화살표 표시.
- 빈 방향만 활성(컬러), 벽/물체 방향은 반투명.
- 화살표 탭 → 해당 방향으로 팽창.

**3) Undo**
- `↶` 버튼 탭 → 즉시 이전 턴으로 (애니메이션 0.2초).
- 길게 누르면 연속 Undo.

**4) 힌트**
- `💡` 버튼은 평소 비활성(회색).
- 스테이지에서 **3분 이상** 진행 없으면 활성화.
- 탭 시 "첫 수 힌트" 한 줄만 노출 (답 전체 X).

**5) 튜토리얼 말풍선**
- 새 메커니즘 등장 스테이지 진입 시 주인공 머리 위 말풍선.
- 대사 1~2줄, 탭하면 사라짐. 재등장 없음.

### 챕터 선택 (월드맵)

- 4개 지역(정원 · 탑 · 광장 · 왕궁)이 한 화면에 보이는 일러스트.
- 클리어 전: 회색 · 흐릿함 / 클리어 후: 파스텔 색 복원.
- 지역 탭 → 해당 챕터의 스테이지 목록 (그리드 2×4 형태).
- 각 스테이지는 **별점(0~3)** + 잠금 상태 표시.

### 접근성

- **색맹 대응**: 물=파랑+물결 패턴, 얼음=하늘색+결정 패턴, 모닥불=주황+불꽃 아이콘 → 색 외 형태로도 식별 가능.
- **글자 크기**: 본문 20 px 이상, 말풍선 24 px.
- **한글 폰트**: `Pretendard` 또는 `Nanum Gothic` (가독성).
- **사운드 OFF 토글**: 수업 중 음소거 필수.
- **자막**: 컷신 대사는 항상 텍스트 동반 (사운드 꺼도 이해 가능).

### 설정 메뉴

- 사운드 볼륨 (BGM / SFX 각각)
- 진도 초기화 (비밀번호 확인)
- 크레딧

## 7. 배포 · 개발 로드맵 · 범위

### 배포 전략

- **주 배포**: itch.io (인디 게임 신뢰감, 링크 공유 편리).
- **미러**: GitHub Pages 자체 호스팅 (커스텀 URL).
- **한국 교사 커뮤니티**: 인디스쿨 자료실, 에듀넷 등에 링크 공유.
- 전부 **정적 호스팅** (서버 없음).
- **파일 크기 목표**: 초기 로드 **5 MB 이하** (교실 와이파이 고려).

### 개발 로드맵 (단계 순서 중요)

**Phase 0 — 프로젝트 부트스트랩 (1주)**
- Vite + Phaser 3 + TypeScript 셋업.
- `core/Grid`, `TurnEngine`, `UndoStack` 순수 로직 + vitest 테스트.
- 최소 렌더링 (임시 도형 타일).

**Phase 1 — 기본 메커닉 (2주)**
- 이동 · 물 붓기 · 얼리기 · 녹이기 · 팽창 · 미끄러짐.
- 레벨 YAML 로더.
- 간단 HUD.
- 5개 테스트 스테이지 플레이 가능.

**Phase 2 — 챕터 1 + 아트 통합 (2주)**
- 주인공 · 타일 · 오브젝트 스프라이트 (무료 CC0 에셋 활용 후 리터치).
- 파스텔 팔레트 확립.
- Ch1 스테이지 8개 완성.
- **1반 파일럿 테스트** (4학년 실제 플레이).

**Phase 3 — 저울 + Ch2~4 (3주)**
- 양팔 · 게이트 · 시소 3형태 구현.
- Ch2~4 스테이지 총 28개 작성 (Claude로 초안 생성 → 직접 플레이 검증).
- 솔버 스크립트로 "풀이 가능성" 자동 검증.

**Phase 4 — 컷신 · 사운드 · 폴리시 (2주)**
- 챕터 시작/엔딩 컷신 일러스트 + 간단 애니.
- 색 복원 연출 (color tween).
- 효과음(무료 라이브러리) + BGM 4곡 (챕터별).
- 튜토리얼 말풍선, 힌트, 별점 모달.

**Phase 5 — QA + 배포 (1주)**
- 2~3반 파일럿 재테스트, 피드백 반영.
- itch.io + GitHub Pages 배포.
- 교사 커뮤니티 공유 글 작성.

**총 예상**: **약 11주** (개인 개발, 주말·저녁 작업 가정).

- ❌ 레벨 에디터 · 유저 생성 콘텐츠
